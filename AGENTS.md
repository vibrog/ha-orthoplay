# Agent Guide – ha-orthoplay

Home Assistant custom integration for the **Teenage Engineering OD-11 wireless speaker**. The integration connects via a persistent WebSocket on port 8081 and was reverse-engineered from the Orthoplay web application. IoT class: `local_push`.

---

## Architecture

### Data flow

```
OD11Client (client.py)
  └─ WebSocket ──► speaker state pushed in real-time
  └─ state{}     ──► read by all entity files
  └─ device{}    ──► read by all entity files
  └─ callbacks[] ──► entities register here; fired on every state change
```

There is **no DataUpdateCoordinator**. All updates are push-based. Entities register a callback with the client (`client.register_callback(fn)`) and call `self.async_write_ha_state()` inside it.

### Startup sequence

1. `__init__.py: async_setup_entry` creates `OD11Client`, calls `client.connect()`.
2. Client sends `global_join` → receives `global_joined` with MAC and firmware revision.
3. Client sends `group_join` → receives `group_joined` with sources and group state.
4. First callback fires; `__init__.py` registers devices in the device registry.
5. Platform setup (`async_setup_entry` in each entity file) adds entities to HA.
6. Per-speaker entities (sensor, binary_sensor, button, update) are created lazily after the first `speaker_added` / `speaker_state_changed` message is received.

## Client API reference

All methods are on `OD11Client` in `client.py`.

### State dicts (read-only from entities)

```python
client.state = {
    "playing": bool,
    "volume": int,          # 0–100
    "volume_max": int,
    "muted": bool,          # True when ALL speakers are muted
    "source": int,          # 0–5
    "sources": list[dict],  # populated after group_join
    "title": str,
    "artist": str,
    "album": str,
    "track_duration": float,          # seconds
    "track_position": float,          # 0.0–1.0 fraction
    "position_updated_at": datetime,
    "eq_bass": bool,
    "eq_mid": bool,
    "eq_treble": bool,
}

client.device = {
    "group_name": str,
    "latest_revision": str,
    "master_mac": str,
    "speakers": {
        mac: {
            "box_serial": str,
            "revision": str,
            "channel": str,   # "left" | "right" | "mono"
            "bt": bool,       # Bluetooth physically connected
            "toslink": bool,  # Optical physically connected
            "linein": bool,   # Line-in physically connected
            "muted": bool,
            "wifi_quality": int,  # 0–100
        },
        ...
    },
}
```

### Commands

| Method | OD-11 action |
|---|---|
| `playback_start()` | `playback_start` |
| `playback_stop()` | `playback_stop` |
| `track_next()` | `track_skip_to_next` |
| `track_prev()` | `track_skip_to_prev` |
| `track_seek(fraction: float)` | `track_seek` (0.0–1.0) |
| `volume_set(vol: int)` | `group_set_volume` (0–100) |
| `volume_up()` | `group_change_volume amount=1` |
| `volume_down()` | `group_change_volume amount=-1` |
| `set_source(src_id: int)` | `group_set_input_source` |
| `enter_standby()` | `group_enter_standby` |
| `set_eq_bass(enabled: bool)` | `group_set_eq_bass_boost` |
| `set_eq_mid(enabled: bool)` | `group_set_eq_mid_boost` |
| `set_eq_treble(enabled: bool)` | `group_set_eq_treble_boost` |
| `playlist_add_url(url: str)` | `playlist_add_url` |
| `playlist_clear()` | `playlist_clear` |

Internal helpers (not called from entities):

- `_send(action, data)` — sends a JSON command over the WebSocket.
- `_connect()` — establishes connection and sends `global_join`.
- `_connection_loop()` — reconnect loop (5 s delay between attempts).
- `_ping_loop()` — keep-alive ping every 20 s.

### Incoming update routing (`_handle_message`)

| `response` / `event` key | Fields updated |
|---|---|
| `group_volume_changed` | `state["volume"]` |
| `playback_state_changed` | `state["playing"]` |
| `group_input_source_changed` | `state["source"]` |
| `track_changed` | `state["title/artist/album/track_duration"]` |
| `track_pos` / `realtime` | `state["track_position"]`, `state["position_updated_at"]` |
| `group_max_volume` | `state["volume_max"]` |
| `group_eq_bass/mid/treble_boost` | `state["eq_bass/mid/treble"]` |
| `speaker_added` / `speaker_state_changed` | `device["speakers"][mac]` |
| `entering_standby` | clears playing state; WebSocket will disconnect |

## Entities

| File | Entity type | Scope | Entity category |
|---|---|---|---|
| `media_player.py` | `MediaPlayerEntity` | Group | — |
| `sensor.py` | `SensorEntity` | Per speaker | `DIAGNOSTIC` |
| `binary_sensor.py` | `BinarySensorEntity` × 3 | Per speaker | `DIAGNOSTIC` |
| `button.py` | `ButtonEntity` | Per speaker | `DIAGNOSTIC` |
| `switch.py` | `SwitchEntity` × 3 | Group | `CONFIG` |
| `update.py` | `UpdateEntity` | Per speaker | `DIAGNOSTIC` |

### Media player capabilities

Features are built dynamically from source `supports_*` flags:

- Always: `TURN_OFF`, `SELECT_SOURCE`, `VOLUME_SET`, `VOLUME_STEP`, `VOLUME_MUTE`
- Per source: `PLAY`, `PAUSE`, `STOP`, `NEXT_TRACK`, `PREVIOUS_TRACK`, `SEEK`, `PLAY_MEDIA`, `CLEAR_PLAYLIST`

`turn_on` is **not** supported (OD-11 wakes via physical button only).  
`pause` maps to `stop` (the device has no separate pause state).

### Unit conversions

| HA convention | OD-11 / internal |
|---|---|
| Volume 0.0–1.0 | 0–100 integer |
| Position in seconds | 0.0–1.0 fraction × duration |
| Duration in seconds | milliseconds ÷ 1000 |

- `_airplay._tcp.local.` with property `manufacturer = teenage engineering`

## Coding conventions

- **No DataUpdateCoordinator** — do not introduce polling; the integration is `local_push`.
- **State access** — read from `client.state[key]` / `client.device[key]` directly inside entities; do not cache locally.
- **Callback pattern** — register with `client.register_callback(self._handle_update)` and call `self.async_write_ha_state()` inside the callback.
- **Unique IDs** — follow the pattern `od11_{entry_id}` (media player) or `od11_{entry_id}_{serial}_{suffix}` (per-speaker entities).
- **Private methods** — `_send`, `_connect`, `_connection_loop`, `_ping_loop` are internal to the client; entities should only call the public command methods.
- **Deferred entity setup** — per-speaker entities are not created until speaker data arrives; match the lazy-init pattern in the existing platform files.
